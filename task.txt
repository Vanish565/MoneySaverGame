    MONEY SAVER GAME — PROJECT NOTES (SO FAR)

    PROJECT GOAL
    Build a budgeting system with gamification features
    Track income and expenses
    Group spending by expenseType
    Show financial summaries
    Later add levels, XP, and game mechanics
    SYSTEM ARCHITECTURE (LAYERED DESIGN)

    ConsoleMenu (UI Layer)
    ↓
    BudgetController (Control Layer)
    ↓
    BudgetTracker (Business Logic Layer)
    ↓
    TransactionRepository (Persistence Layer - JSON)

    RESPONSIBILITIES

    ConsoleMenu:

    Handles user input
    Displays menus
    Validates input (numbers, ranges)
    Calls controller methods

    BudgetController:

    Acts as a bridge between UI and logic
    Routes requests to BudgetTracker
    Keeps UI clean from business logic

    BudgetTracker:

    Owns List<Transaction>
    Adds income and expenses
    Calculates:
    total income
    total expenses
    balance
    spending by expenseType
    Maintains application state

    TransactionRepository:

    Saves transactions to JSON file
    Loads transactions from JSON file
    Uses Gson for serialization/deserialization
    ONLY handles storage (no business logic)
    DESIGN PRINCIPLES LEARNED

    Separation of Concerns:

    UI handles interaction only
    Tracker handles logic only
    Repository handles storage only

    Encapsulation:

    File handling is hidden inside repository
    Other layers do not know storage details

    Dependency Injection:

    BudgetTracker receives repository from outside
    It does not create it internally

    Data Ownership Principle:

    BudgetTracker owns transaction list
    It is responsible for maintaining consistency
